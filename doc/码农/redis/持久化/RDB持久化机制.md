1.创建RDB文件的命令
SAVE 这个会阻塞服务器的进程 直到RDB文件创建完毕为止
BGSAVE 主进程会fork一个子进程进行RDB文件的创建

2.载入RDB文件的时机  
  一旦服务器启动 存在RDB文件就会载入  注意启动redis服务的时候有日志打印
    DB loaded fromdisk：0.13 seconds
  但是注意的是 假如服务器开启AOF功能 那么在服务端启动的时候会载入aof的文件
  
  
3.因为bgsave可以不阻塞主进程的情况下 持久化文件 所以可以通过配置文件进行rdb文件的生成
  save 60 1000 #60秒内如果有1000个key发生变化触发RDB快照的执行
  save 60 1000 #60秒内如果有1000个key发生变化触发RDB快照的执行
  
  底层原理是这样的 对于服务器状态 有一个值保存 保存了服务器自从上次保存后 修改了多少次 并且上一次保存完成的时候的时间 
  然后 定时任务在执行的时候 会判断是不是符合条件 符合条件会执行bgsave命令
